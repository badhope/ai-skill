# PRD Generator

Create a product requirements document from a user's request.

## PRD Structure

Create a Markdown document with these sections:

### 1. Summary
- One paragraph overview
- Problem statement
- Proposed solution

### 2. Requirements
- User stories
- Functional requirements
- Non-functional requirements
- Edge cases

### 3. Design
- UI/UX considerations
- Technical approach
- Dependencies

### 4. Success Metrics
- How to measure success
- KPIs

### 5. Risks & Open Questions

## Process

1. Ask clarifying questions if needed
2. Create PRD in Markdown format
3. Review with user and iterate

## Tips

- Keep it concise but comprehensive
- Use bullet points for scannability
- Clearly distinguish must-have vs nice-to-have
- Include examples where helpful
